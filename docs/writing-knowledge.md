# Engineering.0x1.company 執筆ナレッジ

本ドキュメントは、engineering.0x1.companyのブログ記事執筆における知識とベストプラクティスをまとめたものです。

## 1. 記事の構造とフォーマット

### 1.1 フロントマター
すべての記事は以下の形式のフロントマターを持ちます：

```yaml
---
title: "記事タイトル"
description: "簡潔な説明"
date: "YYYY-MM-DD"
author: tomokisun
---
```

### 1.2 ファイル配置規則
- パス形式：`articles/YYYY/MMDD/filename.mdx`
- 年フォルダ（YYYY）と月日フォルダ（MMDD）で階層化
- ファイル名は英語のケバブケース（kebab-case）を使用
- URLスラッグはファイル名から自動生成

### 1.3 画像の管理
- 画像パス：`/assets/YYYY/MMDD/article-name/image-name.webp`
- 記事ごとに専用フォルダを作成
- WebP形式を推奨（パフォーマンス最適化）
- すべての画像に詳細なalt textを必須で設定

## 2. 執筆スタイルガイド

### 2.1 言語と表記
- **主要言語**：日本語
- **一人称**：「わたしたち」を使用（チーム/企業としての視点）
- **技術用語**：英語のまま記載（Swift、TCA、CloudRun、TypeORMなど）
- **製品名**：正式名称を使用（GitHub Actions、Xcode Cloud、Google Workspaceなど）

### 2.2 文体と語調
- 技術ブログとして専門的かつ親しみやすい文体
- 読者への配慮を持ちながら、技術的な正確性を保つ
- 結論や重要なポイントは明確に示す

### 2.3 記事の流れ
1. **導入部**：背景や課題の説明
2. **本文**：解決策の詳細な説明
3. **実装例**：具体的なコードやコマンド
4. **結果/効果**：導入による改善点
5. **まとめ**：重要ポイントの整理

## 3. MDXコンポーネントの活用

### 3.1 ExternalOgp
外部リンクをリッチに表示するコンポーネント：

```mdx
<ExternalOgp url="https://github.com/0x1-company/ios-monorepo" />
```

使用場面：
- GitHubリポジトリの紹介
- 参考記事へのリンク
- 外部ツールやサービスの紹介

### 3.2 画像の埋め込み
```mdx
<img src="/assets/2025/0323/tech-stack-of-ios/app-screenshots.webp" alt="アプリのスクリーンショット" />
```

画像使用のベストプラクティス：
- 説明的で具体的なalt textを記載
- 技術的な図解には特に詳細な説明を追加
- スクリーンショットは重要な部分を強調

## 4. 技術記事の特徴的なパターン

### 4.1 課題解決型記事
1. **具体的な課題の提示**（エラーメッセージなども含む）
2. **解決策の選定理由**
3. **実装の詳細**（コード例を豊富に）
4. **結果と効果の定量的/定性的評価**

例：CloudRun Jobを使ったDBマイグレーション記事

### 4.2 アーキテクチャ紹介型記事
1. **ビジネス要件の説明**
2. **技術選定の理由**
3. **全体構成の図解**
4. **実装の詳細とコード例**
5. **運用面での工夫**

例：BeMatchのiOS技術スタック記事

### 4.3 自動化・効率化記事
1. **手動作業の課題**
2. **自動化のアプローチ**
3. **ステップバイステップの実装ガイド**
4. **実行例とコマンド**
5. **得られた効果**

例：Google Workspaceグループの作成自動化記事

## 5. コード記載のベストプラクティス

### 5.1 コードブロックの使用
- 言語を明示的に指定（```bash、```typescript、```swift など）
- コンテキストを説明するコメントを追加
- 重要な部分にはインラインコメントで説明

### 5.2 コード例の粒度
- 全体像が分かる程度の適切な長さ
- 省略する場合は「// 設定略...」などで明示
- 実際に動作するコードを心がける

### 5.3 環境設定の明示
```bash

```

## 6. 図表とビジュアル要素

- UIの変更や設定画面を示す際に使用
- 重要な部分が分かりやすいようにクロップ
- 必要に応じて注釈を追加（別途説明文で補足）

### 6.2 アーキテクチャ図

- システム構成図


## 7. 読者への配慮

### 7.1 専門用語の扱い
- 初出時は簡潔な説明を追加
- 略語は正式名称を併記
- 必要に応じて参考リンクを提供

### 7.2 実装の再現性
- 前提条件を明確に記載
- バージョン情報を含める
- 依存関係を明示

### 7.3 エラーハンドリング
- 想定されるエラーと対処法を記載
- トラブルシューティングセクションの追加
- 実際に遭遇したエラーメッセージを含める

## 8. SEOとメタデータ

### 8.1 タイトルの付け方
- 具体的で検索されやすいキーワードを含む
- 技術スタック名を明示
- 解決する課題を示唆

### 8.2 descriptionの書き方
- 記事の要点を1-2文で要約
- 主要なキーワードを自然に含める
- 読者が得られる価値を明確に
